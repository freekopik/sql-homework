# Оптимизация и аналитика базы данных (клон HeadHunter)

Данный репозиторий содержит проект по проектированию, наполнению и оптимизации производительности реляционной базы данных PostgreSQL. В рамках работы реализован полный цикл: от создания схемы до автоматизированного бенчмаркинга индексов.

## 🚀 Описание скриптов автоматизации (`.sh`)

Скрипты обеспечивают управление жизненным циклом проекта и автоматизацию тестирования:

* **`start.sh`** — главный скрипт инициализации. Разворачивает Docker-инфраструктуру, генерирует данные, автоматически запускает `sync_data.sh` для синхронизации контейнеров и проводит финальный `benchmark.sh`.
* **`stop.sh`** — корректно останавливает все контейнеры и очищает временные ресурсы.
* **`sync_data.sh`** — (вызывается из `start.sh`) гарантирует абсолютную идентичность данных в эталонном (без индексов) и оптимизированном контейнерах для честного сравнения.
* **`benchmark.sh`** — (вызывается из `start.sh`) основной инструмент замера производительности, рассчитывающий коэффициент ускорения (**Speedup**) по 18 сценариям.
* **`hashes_comparison.sh`** — инструмент верификации: сравнивает хэш-суммы таблиц между базами, подтверждая, что оптимизация не исказила данные.
* **`db_size.sh`** — выводит детальную статистику по весу каждой таблицы и её индексов. Позволяет оценить "цену" оптимизации и показатель `index_ratio`.
* **`run_analytics.sh`** — автоматический запуск набора бизнес-отчетов для проверки корректности агрегации данных.

## 📊 Аналитический блок (`.sql`)

Реализованы SQL-скрипты для извлечения ценных бизнес-метрик:

* **`get_avgs_for_area.sql`** — расчет средних зарплат по регионам. Включает логику обработки неполных данных (вилки "от" и "до") через `COALESCE`.
* **`get_top_vacancies.sql`** — выявление востребованных вакансий, получивших более 5 откликов за первую неделю после публикации.
* **`get_top_months.sql`** — анализ сезонности найма и определение пиковых месяцев по количеству вакансий.



## 🏗 Инфраструктура и оптимизация

* **`schema.sql`** — описание логической структуры (таблицы, связи, констрейнты).
* **`test_data.sql`** — генератор 1 млн+ записей. Использует систему региональных коэффициентов зарплат для создания реалистичного распределения данных.
* **`indexes.sql`** — реализация стратегии индексирования (B-Tree для фильтров и GIN для текстового поиска).
* **`samples_for_indexes.sql`** — набор из 18 тестовых сценариев, эмулирующих реальную нагрузку HR-портала.



## 📈 Результаты оптимизации

По результатам работы `benchmark.sh`:
* Максимальное ускорение на сложных выборках (поиск текущего места работы, фильтрация опыта) достигает **800x - 900x**.
* На сверхмалых справочниках (размером в 1 страницу памяти) зафиксирован `Speedup < 1.0x`. Это объясняется тем, что планировщику Postgres выгоднее выполнить `Sequential Scan`, чем тратить ресурсы на чтение структуры индекса.

## 🛠 Инструкция по запуску
1. Просто запустите главный скрипт: `./start.sh`
2. По завершении (примерно 3-5 мин) вы получите готовый отчет Benchmark в консоли.
3. Для получения аналитики: `./run_analytics.sh`
4. Для проверки веса индексов: `./db_size.sh`
